Это пример по запуску сервера и вывода результатов из него.

Для его работы, на компьютере должен быть установлен Node.js. Скачать его можно с https://nodejs.org/ . Лучше брать LTS версию.
Для компьютеров на Windows 7 не подойдет последняя LTS версия Node.js. Необходимо устанавливать более старую (например, https://nodejs.org/download/release/v13.6.0/).

Необходимо открыть папку для сервера и в файле *package.json* выполнить скрипт, помеченный `install-dependencies`. Это можно сделать с помощью дополнений для VS Code, либо прописав в командной строке (запущунной из той папки, где находится *package.json*): **`npm run-script install-dependencies`**.
После этого будет создан *package-lock.json* файл со списоком установленных зависимостей, а сами установленные пакеты будут лежать в папке *node_modules*.

Необходимая зависимость - пакет *json-server*. Он позволяет запускать сервер из базы данных, записанной в .json файл, и обрабатывать запросы по REST архитектуре. В этом примере база находится в файле db.json. Запустить сервер можно по скрипту `run-server`: **`npm run-script run-server`**. Скрипт будет жить, пока не будет прерван (например, закрытием командной строки, в которой он открыт).

В этом примере в базе существует только таблица `persons`. После запуска сервера, клиент может отправлять запросы на адрес http://localhost:3000/persons.

В папке с клиентом находится `index.html` файл, подключающий `index.js` скрипт, который пользуется запросами с сервера.

Отладить клиент в VS Code можно, если существует конфигурация запуска в файле `.vscode/launch.json`. В качестве цели отладки будет использоваться местный файл `index.html`, отладка будет вестись в бравзере Google Chrome. Можно использовать и другие бравзеры, если сменить значение поля `type`. Например, `edge` вместо `pwa-chrome`, чтобы использоваться Edge вместо Google Chrome.
